# 统一运维管理平台一期 - 任务拆解与开发计划

## 开发阶段总览

| 阶段 | 内容 | 里程碑 |
|------|------|--------|
| M0 | 项目初始化与基础架构搭建 | 已基本实现：项目骨架可运行 |
| M1 | 公共基础模块（认证/权限/审计） | 已基本实现：用户可登录并受权限控制 |
| M2 | CMDB 配置管理模块 | 已基本实现：资产可录入、查询、API 同步 |
| M3 | 监控系统模块 | 未开始：指标可采集、存储、可视化展示 |
| M4 | 告警平台模块 | 未开始：告警可触发、通知、处理闭环 |
| M5 | 联系人管理与工单流程管理 | 未开始：告警→通知→工单→闭环全流程打通 |
| M6 | 集成测试、性能测试、UAT | 未开始：达到上线标准 |
| M7 | 部署上线与运维交接 | 未开始：生产环境稳定运行 |

## 任务依赖关系

```
M0（基础架构）
 └──→ M1（公共基础）
       ├──→ M2（CMDB）
       │     ├──→ M3（监控，依赖 CMDB 资产关联）
       │     │     └──→ M4（告警，依赖监控数据）
       │     └─────────→ M5（工单流程管理，依赖 CMDB 负责人查找）
       └───────────────→ M5（联系人管理，依赖用户体系）
M4 + M5 完成后 ──→ M6（集成测试）
M6 完成后 ──→ M7（部署上线）
```

**可并行的任务：**
- M2 和 M1.3（审计日志）、M1.4（系统配置）可并行
- M3.1（数据采集）和 M3.3（仪表盘前端）可并行
- M5.1（联系人管理）和 M4（告警平台）可并行
- M6.1（单元测试）与各模块开发可同步进行
