# 统一运维管理平台一期 - 测试验收清单

## 一、公共基础模块验收

### 1.1 用户认证
- [ ] 用户可通过用户名+密码登录系统，登录后获取 JWT Token
- [ ] 密码使用 bcrypt 加密存储，数据库中无明文密码
- [ ] Token 过期后自动拒绝访问，返回 401
- [ ] Token 刷新机制正常工作，无感续期
- [ ] 登出后 Token 失效，无法继续访问
- [ ] 连续登录失败有防暴力破解机制（限流/锁定）
- [ ] LDAP/AD 认证接口已预留（AuthProvider 接口定义）

### 1.2 RBAC 权限管理
- [ ] 系统管理员可管理所有功能（用户/角色/系统设置）
- [ ] 运维管理员可管理 CMDB 模型、监控策略、告警规则
- [ ] 运维操作员可查看/编辑资产、查看监控、处理告警
- [ ] 只读用户仅可查看，无法执行任何修改操作
- [ ] 无权限操作返回 403，并记录审计日志
- [ ] 资源级权限控制生效（按资产分组控制数据可见范围）

### 1.3 审计日志
- [ ] 所有 API 操作自动记录审计日志（操作人、时间、操作类型、资源、变更内容）
- [ ] 审计日志支持按操作人、操作类型、时间范围筛选
- [ ] 审计日志不可修改、不可删除
- [ ] 审计日志查询页面分页正常

### 1.4 系统配置
- [ ] 系统配置可通过 API 读写
- [ ] 配置变更实时生效（缓存更新）

## 二、CMDB 配置管理模块验收

### 2.1 模型组与模型管理
- [ ] 可创建自定义模型组（输入名称、描述、图标）
- [ ] 可在模型组下创建模型，定义字段（名称、类型、是否必填、默认值）
- [ ] 支持所有字段类型：文本、数字、枚举、日期、IP 地址、关联引用
- [ ] 字段校验规则生效（必填校验、枚举值范围、正则表达式、数值范围）
- [ ] 可定义模型间关系（如：服务器 → 运行 → MySQL）
- [ ] 预置模型数据完整加载（物理服务器/虚拟机/交换机/路由器/Nginx/MySQL/Redis/业务应用）
- [ ] 模型组/模型支持编辑和删除（有资产引用时删除需提示）
- [ ] 前端根据模型字段定义动态渲染表单

### 2.2 资产管理
- [ ] 可通过表单手工录入资产，字段与模型定义一致
- [ ] 资产列表支持按模型类型、分组、状态、关键字筛选
- [ ] 资产列表支持分页，展示核心字段
- [ ] 资产详情页展示完整属性信息
- [ ] 资产分组管理正常（按业务线/环境/地域分组）
- [ ] 资产生命周期状态可切换（采购→上架→运行→维修→报废）
- [ ] 资产每次修改自动记录变更历史（变更前后值、操作人、时间）
- [ ] 资产变更历史可查询和回溯

### 2.3 资产批量导入
- [ ] 支持 Excel 文件上传导入
- [ ] 支持 CSV 文件上传导入
- [ ] 导入时进行字段校验，校验失败的行返回错误明细
- [ ] 导入结果展示成功/失败数量

### 2.4 API 采集
- [ ] POST /api/v1/cmdb/assets/sync 接口正常工作
- [ ] 支持增量同步模式（按 unique_key 更新）
- [ ] 支持全量同步模式
- [ ] API Key + Secret 签名认证生效，无效认证返回 401

### 2.5 资产详情页关联展示

- [ ] 资产详情页展示当前活跃告警

## 六、通知分发验收

### 6.1 通知路由规则
- [ ] P0（紧急）告警触发时，通过电话 + 短信 + 邮件三种渠道通知负责人
- [ ] P1（重要）告警触发时，通过短信 + 邮件两种渠道通知负责人
- [ ] P2（一般）告警触发时，通过邮件渠道通知负责人
- [ ] P3（提醒）告警通过邮件通知（可选）

### 6.2 通知对象查找
- [ ] 告警触发后，系统正确从 CMDB 查找告警设备的负责人
- [ ] 根据负责人查找关联的联系人及其通知渠道配置
- [ ] 设备无负责人时，通知发送给默认运维组

### 6.3 通知渠道发送
- [ ] 邮件通知正常发送（SMTP），内容包含告警详情
- [ ] 短信通知正常发送（对接短信网关），内容包含告警摘要
- [ ] 电话通知正常发送（对接语音通知 API），语音播报告警信息
- [ ] 企业微信 Webhook 通知正常发送
- [ ] 钉钉 Webhook 通知正常发送

### 6.4 通知模板
- [ ] 通知模板支持变量替换（告警名称/资产名/IP/等级/时间/指标值）
- [ ] 通知内容格式正确，信息完整

### 6.5 通知记录
- [ ] 每次通知发送记录保存成功（接收人/渠道/内容/发送状态/时间）
- [ ] 通知记录查询页面正常（筛选+分页）
- [ ] 发送失败的通知有重试机制

## 七、工单流程管理验收

### 7.1 工单自动生成
- [ ] 告警触发并发送通知后，系统自动创建运维工单
- [ ] 工单包含：告警信息、告警等级、关联资产信息（含负责人）、状态=待处理
- [ ] 同一告警不重复创建工单（去重逻辑）

### 7.2 工单状态流转
- [ ] 工单状态可正常流转：待处理 → 处理中 → 已完成 → 已闭环
- [ ] 状态流转有权限控制（仅处理人或管理员可操作）
- [ ] 每次状态变更记录操作人和时间

### 7.3 工单处理记录
- [ ] 运维人员可在工单中添加处理记录（文字描述）
- [ ] 处理记录按时间线展示
- [ ] 处理记录不可修改、不可删除

### 7.4 工单闭环与告警关闭
- [ ] 工单标记为"已闭环"后，关联告警状态自动更新为 Closed
- [ ] 闭环时间正确记录
- [ ] 工单闭环后不可再添加处理记录

### 7.5 工单视图
- [ ] 工单列表支持按状态/等级/资产/负责人筛选
- [ ] 工单详情页展示完整信息（告警信息+资产信息+处理记录时间线+状态操作）
- [ ] 运维经理可查看团队所有工单和处理情况

### 7.6 工单统计
- [ ] 可查看未闭环工单数量
- [ ] 可查看平均处理时长
- [ ] 可按处理人统计工单处理量
